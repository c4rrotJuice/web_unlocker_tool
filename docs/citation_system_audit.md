# Citation System Audit (Extension + Web App)

## Current pipeline before refactor

1. Page content is scanned in content scripts (`extension/content/unlock_content.js`) and injected web script (`app/static/unlock.js`).
2. Minimal metadata is extracted from a few fields only:
   - title (`og:title` / `twitter:title` / `document.title`)
   - site (`og:site_name` / app name / hostname)
   - author (`meta[name=author]` / `article:author` / first JSON-LD author)
3. Citation strings were generated by one generic formatter function per app with hardcoded templates and current year.
4. `Copy MLA/APA/Chicago/Harvard` copied whatever preview text was shown (extension) or re-formatted on copy (web app).
5. Formatting happened before clipboard write, but metadata quality and style rules were inconsistent.

## Weaknesses identified

- No layered fallback extraction for microdata, semantic HTML, or byline containers.
- Publication year defaulted to current year instead of parsed publication date.
- Multi-author handling missing.
- Corporate/organizational author handling missing.
- APA/Harvard initials and MLA/Chicago punctuation were not style-compliant.
- Paragraph quote locators were inconsistent and not style-specific.
- Missing validation/fallback layer when title/site/author/date is absent.
- Site-name/domain fallback could return lowercase or partial values.
- Web app and extension formatting diverged over time.

## Refactor outcomes

- Added normalized metadata extraction with ordered fallback strategies.
- Added metadata validation and graceful fallback logic.
- Added style-specific formatter behavior for MLA/APA/Chicago/Harvard.
- Added quote-aware in-text references with optional paragraph locators.
- Unified copy flows to ensure fully formatted citation strings are copied.

## Metadata-first citation architecture (2026-03)

- Citation metadata is now normalized server-side and treated as the source of truth.
- The API derives and persists two explicit outputs per style:
  - `inline_citation` (in-text form)
  - `full_citation` (bibliography/reference entry)
- `full_text` is retained for backward compatibility and mirrors `full_citation` when missing.
- The extension/web injector now submit structured metadata plus explicit inline/full outputs so the editor can insert in-text citations separately from bibliography entries.


## Metadata-first hardening update (this change)

- Backend now computes a stable `source_fingerprint` (DOI -> canonical URL -> metadata hash) and `source_version` from normalized metadata.
- Citation metadata now includes explicit `source` and `quote` objects so source identity and quote instances are separated without breaking existing API contracts.
- Backend precomputes a lightweight `render_cache` map (style + output + source version) while treating metadata as canonical.
- New `POST /api/citations/render` endpoint supports deterministic citation regeneration from stored metadata for style switching.
- Extension and web app now request server-rendered citations for standard styles, reducing formatter divergence across clients.


info:
  title: Writior V2 API
  version: 2.0.0
  summary: Strict implementation-ready API contract for Writior V2
  description: |
    Canonical backend API for the Writior V2 research-to-writing product.

    Implementation rules:
    - All protected endpoints require Authorization: Bearer <supabase_access_token>
    - No JS-readable auth cookie trust
    - One canonical serializer per entity family
    - All list endpoints use cursor pagination unless explicitly noted
    - Replace-all relation writes must be atomic
    - Extension routes orchestrate canonical services only
    - Seeded extension->editor flows are first-class, not special cases
    - All timestamps are ISO 8601 date-time strings in UTC
    - All ids are UUIDs
    - All success/error responses use the shared envelope contract

servers:

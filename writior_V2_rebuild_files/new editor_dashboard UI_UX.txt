# v2 Frontend UI/UX Implementation Brief

## Objective

Design and implement the frontend interface for both the **Dashboard** and **Editor Workspace**.

The UI must be:

* lightweight
* visually calm and professional
* fast to load
* keyboard friendly
* research‑workflow oriented

Lyra is **not a generic note‑taking tool**. The UI must reinforce the research pipeline:

Sources → Citations → Quotes → Notes → Documents

Every part of the UI should guide the user through this flow.

---

# Core UI Principles

## 1. Lightweight Architecture

Avoid heavy frameworks.

Preferred stack:

* HTML templates
* modular vanilla JS
* AlpineJS for small interactions
* HTMX for server actions

Do NOT introduce React/Next/Vue unless absolutely necessary.

---

## 2. Layout Philosophy

The application uses a **three‑panel workspace model**.

Global layout pattern:

Sidebar | Main Content | Context Panel

This keeps navigation stable while allowing deep research interactions.

---

Research
Documents
Projects

Insights

# Dashboard Interface

The dashboard acts as the user's **research command center**.

Layout:

Sidebar | Dashboard content

## Dashboard Sections

Greeting header

Research activity summary:

* Unlocks today
* Sources captured
* Notes written
* Documents updated

Recent Research:

* Recently captured sources

Recent Documents:

* Recently edited documents

Milestones:

* Research streak
* Sources captured achievements

Use card based layout for these blocks.

---

# Research Explorer

The Research section manages the knowledge graph.

Subsections:

Sources
Citations
Quotes
Notes

Each list should display:

Title
Author
Date captured
Connected notes
Connected quotes

Clicking a source opens the Source Detail panel.

---

# Source Detail View

Displays the full context of a research source.

Sections:

Source metadata
Citation preview
Quotes extracted from the source
Notes linked to the source
Documents using the source

This reinforces the research‑to‑writing workflow.

---

# Projects Interface

Projects organize research collections.

Project list view:

Project name
Color
Last opened

Project detail shows:

Sources
Notes
Documents

---

# Insights Page

The Insights page visualizes research patterns.

Possible components:

Top research domains
Citation style breakdown
Research activity heatmap
Monthly research summary

Charts should be simple and lightweight.

---

# Editor Workspace

The editor is the **core product interface**.

Layout:

Research Explorer | Writing Editor | Context Panel

---

## Left Panel — Research Explorer

Contains quick access to:

Sources
Quotes
Notes
Citations

Selecting an item reveals related research data.

---

## Center Panel — Writing Editor

Main document editing area.

Features:

Title field
Rich text editing
Inline citations
Quote insertion
Note references
Autosave
Checkpoints
Outline navigation

Editor must feel distraction free.

---

## Right Panel — Context Tools

Dynamic panel responding to user actions.

Examples:

If text selected:

Insert citation
Insert quote
Attach note
Find related sources

If cursor inside citation:

Citation preview
Edit citation
Open source

If nothing selected:

Document outline
Attached sources
Attached notes
Tags
Export options

---

# Document Outline

Auto generated from document headings.

Allows quick navigation between sections.

---

# Citation Insertion UX

Typing @ triggers citation search.

Autocomplete list appears.

Selecting a citation inserts formatted reference.

---

# Notes UX

Notes act as side thoughts linked to research.

Inline note markers may appear inside the editor.

Hovering reveals full note content.

---

# Extension Integration

Browser extension integrates directly with Lyra.

When text is highlighted on a webpage, a capture pill appears:

Save note
Save quote
Cite source
Work in editor

Selecting "Work in editor" should:

Create source record
Create citation instance
Create document
Open editor with inserted quote

This workflow must feel instant.

---

# Performance Principles

The UI must feel extremely responsive.

Requirements:

Lazy load heavy panels
Avoid unnecessary re‑renders
Use lightweight JS modules
Prefer server driven UI updates

---

# Microinteractions

Small UI touches improve perceived quality.

Examples:

Smooth panel transitions
Autosave indicators
Hover highlights
Citation preview popovers
Achievement notifications
